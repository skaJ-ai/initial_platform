# Context — 외부 환경

> v3 가설이 도출된 외부 조건들. 이 조건이 바뀌면 가설도 재검토해야 함.

---

## 1. 7월 엔터프라이즈 LLM 도래

### 1.1 사실관계

- 2026-07부터 사내망 안에 엔터프라이즈 LLM(KODEX, Claude, Gemini)이 들어옴
- 모델뿐 아니라 워크스페이스/지식 업로드/툴콜링 등 제품 단위로 들어옴
- 도입 주체: AX팀 (People팀과 무관)
- 사내망 동거인이라 "데이터 격리"만으로는 차별 안 됨

### 1.2 엔터프라이즈가 가져오는 것 (추정)

대부분의 엔터프라이즈 등급은 다음을 포함:
- SSO/SCIM 연동
- 권한 통제 (RBAC, 프로젝트 단위)
- Audit log
- Retention 0 옵션
- SOC2 / ISO27001 / HIPAA 준수
- 워크스페이스 / 지식 업로드 / 툴콜링 / 검색 통합

즉 일반 거버넌스(권한/감사/데이터 retention)는 같이 들어옴. 살아남는 차별은 **HR 도메인 특화 정책 영역**.

### 1.3 100% 안전한가? — 아니다

엔터프라이즈 도입돼도 다음 누수 채널은 남는다:
1. 추론 시 일시 캐시/메모리 잔존
2. 벤더 운영 인력 인시던트 대응 시 접근
3. 컴플라이언스 보존 의무 (법적 요구)
4. Prompt injection 누수
5. 국가 단위 데이터 요청 (예: US CLOUD Act)

→ "사내망에 들어옴 = 벤더 절대 못 봄"은 동치가 아니다. 임원 인사 결정, 노조 관련, 징계, 신원특정 가능 평가 코멘트 같은 것은 엔터프라이즈에 못 보낸다.

## 2. Gauss vs 엔터프라이즈 — 보안/성능 trade-off

### 2.1 Gauss (사내 sLM)

- 오픈소스 모델 파인튜닝 기반 (삼성 Gauss)
- 모델 가중치 + 추론 + 로그까지 사내 통제
- vendor-side 로그 안 남기는 운영 가능
- 데이터 주권 100%, 벤더 의존도 0
- **약점**: 사외 프론티어 대비 성능 약함 (특히 복잡 추론, 긴 컨텍스트, 멀티모달)

### 2.2 엔터프라이즈 (KODEX / Claude / Gemini)

- 프론티어급 성능
- 일반 거버넌스 (SSO / Audit / Retention 등) 같이 들어옴
- **약점**: 5가지 누수 채널, 벤더 관할권 (외국 법 적용 가능)

### 2.3 그래서 라우팅 정책

| | 단순 작업 | 복잡 추론 |
|---|---|---|
| 비민감 | Gauss OK | Enterprise |
| 중간 민감 | Gauss | **People팀 정책 결정 ★** |
| 극민감 | Gauss | Gauss (성능 손해 감수) |

별표 셀이 People팀의 자리다. 마스킹/익명화/요약/분리 호출의 정책이 여기서 정의된다.

### 2.4 Audit의 책임 분담

- vendor-side 로그: Gauss는 안 남김 (보안 강점), 엔터프라이즈는 남길 수도 있음
- application-side 로그: 양쪽 다 우리 플랫폼이 잡는다 (책임 분담 명확)

→ "Gauss + 우리 플랫폼 audit"가 보안+감사 동시 충족의 유일한 조합.

## 3. AX팀 vs People팀 영역 분리

### 3.1 영역 정의 (정치적으로 가장 중요)

| 영역 | 주체 |
|---|---|
| 엔터프라이즈 LLM 도입 / 운영 | AX팀 |
| 사내 sLM (Gauss) 인프라 | AX팀 또는 별도 인프라팀 (확인 필요) |
| 일반 AI Gateway / 라우팅 인프라 | AX팀 |
| **HR 도메인 민감도 분류 정책** | **People팀 (우리)** |
| **HR 도메인 워크플로우 / 승인 라인** | **People팀 (우리)** |
| **HR 데이터 거버넌스 정책** | **People팀 (우리)** |

### 3.2 충돌 회피 프레임

데모/포지셔닝 시 다음 프레임 일관 유지:
- ❌ "엔터프라이즈를 대체합니다"
- ⭕ "AX팀이 깐 인프라 위에 People팀 도메인 정책이 얹힙니다"
- ⭕ "엔터프라이즈가 들어올수록 우리 플랫폼도 좋아집니다 (BYOM)"

AX팀 영역 침범 인상 절대 금지. AI Gateway 인프라를 우리가 build하면 충돌. **우리는 정책 API의 단일 책임 부서**라는 위치 고정.

### 3.3 People팀 내부

- 정보보호센터가 People팀 소속 → 강제 정책 합의에 결정적 호재
- People팀장이 "하라"고 하면 부서 차원 협력 가능
- AXN 컬쳐이노베이션 랩이 People팀장에게 정책 제안 올릴 수 있는 위치
