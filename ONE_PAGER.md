## 두 축

### ① 상용 LLM의 안전한 관문 — 진입의 정당성

데이터 등급(기밀 / 민감 / 일반)에 따라 처리 경로가 분기됨.
민감 등급은 비식별화 게이트가 사외 송출 시 차폐, 응답 수신 시 사내 재매핑.
매핑 테이블은 사내에만 보관. 외부엔 차폐된 업무 맥락만 전달.

**메시지**: "**민감 정보는 사외로 나가지 않습니다**"

### ② 검수형 자산화 — 시간이 우리 편

Work Card 사용에서 누적된 노하우·판단이 Tacit Candidate로 큐잉되고, Domain Experts 검수를 거쳐 ③ 자산 데이터에 등재.
개인 → 셀(조직×도메인) → 도메인 종방향(부문) → 전사 표준 단계 승급.

**메시지**: "**쓸수록 People 조직의 지식 자본이 됩니다**"

## 데이터 등급 × 처리 경로

| 등급 | 처리 | 모델 |
|---|---|---|
| **기밀** | 사외 송출 금지 | Gauss (사내, 로그 없음) |
| **민감** | 비식별화 게이트 후 외부 | 상용 LLM (Claude / Gemini / KODEX) |
| **일반** | 그대로 외부 | 상용 LLM |

**자동 분류 3계층**: 셀(조직×도메인) 기본 등급 → 키워드/패턴 PII detection → 사용자 동의(상향만 가능, 하향 불가).

## 단순 챗 + Skill만으로 안 되는 이유

> "외부 LLM에 Skill 라이브러리만 주면 끝 아닌가?" 라는 질문에 대한 답.

| 요구사항 | 단순 챗 + Skill 라이브러리 | HR AX Platform |
|---|---|---|
| HR 데이터 등급(기밀/민감/일반) 자동 분류 | ❌ Skill엔 등급 개념 없음 | ⭕ 셀×도메인 + 패턴 + 사용자 동의 3계층 |
| 기밀 데이터를 외부에 안 보내기 | ❌ Skill에 데이터를 넣는 순간 사외 송출 | ⭕ 기밀은 사내 Gauss 전담 |
| 민감 데이터 비식별화 후 외부 사용 | ❌ Skill엔 차폐 로직 없음 | ⭕ 비식별화 게이트 + 사내 재매핑 |
| 셀(조직×도메인)별 다른 차폐 정책 | ❌ Skill md엔 정책 못 담음 | ⭕ 셀별 차폐 정책표 |
| 같은 셀 다른 담당자 자산 공유 | ❌ 콜드 스타트 매번 처음부터 | ⭕ Workbench가 자산 retrieval |
| 부문이 산하 사업부 셀 자산 열람 | ❌ Skill엔 조직 계층 개념 없음 | ⭕ 도메인 종방향 승급으로 자동 |
| 비정형 → 정형 승급 | ❌ Skill은 정적 md, 진화 안 함 | ⭕ 반복 패턴이 신규 L6로 승격 |
| Domain Experts 검수 워크플로 | ❌ 사용자가 알아서 | ⭕ 승인/반려/보류 3분기 검수 |
| 매핑 사내 보관 / Audit | ❌ 외부 LLM 책임 | ⭕ Application-side audit, 매핑 사내 |
| 표준 데이터 자동 적용 | ❌ Skill에 매번 복붙 | ⭕ Lab 운영 표준 데이터 자동 RAG |
| 인사 DB 권한별 접근 차등 | ❌ Skill엔 RBAC 없음 | ⭕ Workbench 권한이 DB 접근 게이트 |
| 강제 사용 정당성 | ❌ "그냥 Skill 좋네" | ⭕ "안전 관문 통과 안 하면 누설 사고" |

**결론**: 외부 LLM + Skill 라이브러리는 **점**. 우리 Platform은 그 점들을 **셀(조직×도메인) 좌표 + Workbench + 검수 파이프라인**으로 묶는 격자. 점은 격자가 없으면 흩어진다.

## 데이터 3종 (플랫폼이 적재)

### ① 운영 데이터 (Master)
시스템이 갖춰야 할 최소 데이터. 로그인·권한 관리를 위한 사용자 식별 정보, 조직·개인·발령 등 플랫폼 자체 기능 수행에 필수적인 마스터 데이터. DW가 아닌 연계용 DB에서 1시간 단위로 동기화.

### ② 표준 데이터 (Standards)
부서·전사 차원에서 공식적으로 운영되는 자료. 인사 제도, 규정, 업무 프로세스(L4/L5/L6), 표준 양식 등. AX&CI Lab이 운영하며 잦은 수정·생성을 직접 관리. 비정형 자료를 LLM이 이해하도록 각주·설명을 첨부하여 RAG·벡터화·학습에 사용.

### ③ 자산 데이터 (Knowledge)
HR 담당자의 사용 흔적이 누적되어 형성되는 People 조직 자산. Workbench의 Work Card 사용에서 발생한 노하우·판단·결정이 Tacit Candidate로 큐잉되고, Domain Experts 검수를 거쳐 자산 데이터에 등재. 셀(조직×도메인) 좌표와 공개 범위(개인 / 셀 / 도메인 종방향 / 전사 표준)를 부착.

## Function Calling 패턴 (외부 LLM과 도구의 관계)

```
1. 플랫폼이 외부 LLM에 Tool 목록 + Skill 컨텍스트 묘사 전달
   (Tool 코드 자체는 안 보냄)
2. 외부 LLM이 "Tool X를 인자 Y로 호출" 의도 반환
3. 플랫폼이 Tool 실제 실행 (사내)
4. 결과를 비식별화 게이트 통과 후 외부 LLM에 컨텍스트로
5. 외부 LLM이 최종 응답 생성 → 사내 재매핑 → 사용자
```

**핵심**: 외부 LLM은 의도만 표현. 실제 실행·데이터 노출 통제는 플랫폼이 전담.
