## 우리가 만들 것

**HR 담당자가 HR 데이터를 다룰 때 거쳐야 하는 단일 진입점.**
민감도에 따라 사내 Gauss 또는 사외 엔터프라이즈로 자동 라우팅하고, 사용할수록 회사 맥락이 자산으로 누적되는 운영 레이어.

## 왜 지금 이게 필요한가

2026-07부터 사내망에 엔터프라이즈 LLM(KODEX/Claude/Gemini)이 들어온다. 같은 사내망에 좋은 모델·챗 UI·Skill 라이브러리가 깔린다는 뜻이다.

그 뒤로 일어날 일을 가정해 보자.

- HR 담당자가 임직원 정보를 사외 LLM에 직접 입력 → **데이터 누수 사고 시간 문제**
- "어차피 엔터프라이즈가 더 똑똑하니까 그거만 쓴다" → 우리 플랫폼 차별 인식 0
- 11월에 우리 플랫폼 출시해도 "Claude Enterprise랑 뭐가 달라요?"에 답 못 함

## 우리가 답하는 방식 (3 메커니즘)

| | 메커니즘 | People팀이 가진 카드 |
|---|---|---|
| ① **Mandate** | HR 데이터 처리는 우리 플랫폼 통과 필수 | 정보보호센터가 People팀 소속 |
| ② **Routing** | 민감 데이터는 Gauss로, 비민감은 엔터프라이즈로 자동 분기 | HR 도메인 민감도 분류는 People팀만 정의 가능 |
| ③ **Accumulation** | 사용 흔적이 검수 거쳐 회사 자산으로 승급 | Process Coaching L4/L5/L6 자산 + AXIOM 설계 |

## 엔터프라이즈가 절대 카피 못 하는 이유

- 강제는 **정책**: People팀만 정당성 있음
- 분류는 **HR 도메인 지식**: AX팀도, 벤더도, 일반 AI Gateway도 못 만듦
- 누적은 **회사 맥락**: 시간이 갈수록 강해짐, 5년 후엔 카피 불가능

## 데모 시나리오 (11월 출시 시점)

| 사용자 요청 | 플랫폼 동작 |
|---|---|
| "다음달 인사위 안건 정리해줘" | 임원 인사 데이터 포함 감지 → **Gauss 처리** |
| "올해 채용시장 트렌드 정리해줘" | 비민감 + 복잡 분석 → **엔터프라이즈 처리** |
| "임원 A씨 평가 코멘트 기반으로 강점·약점 분석해줘" | 민감 + 복잡 → **익명화 후 엔터프라이즈** + 마스킹 부분 시각화 |

모든 호출은 application-side audit log로 추적. 모든 결정은 회사 자산으로 누적.

## 부서장께 요청드리는 결정 3가지

1. **v3 방향성 승인** — 현재 "스킬+툴콜링" 빌드를 v3로 재정렬할 권한
2. **정보보호센터 협의 트리거** — 분류표 v0 합의 카운터파트 지정
3. **AX팀과의 영역 분리 합의 후원** — "AX팀 인프라 위에 People팀 도메인 정책" 프레임
