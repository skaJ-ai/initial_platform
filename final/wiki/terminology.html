용어 사전

PM 친화 한국어 + 영문 원어 병기. 부문장·임원·다부서 청중 모두에게 통하는 표현으로 정리.

실행 레이어

용어본질설명
Agent LLM 처리 단위 (판단·결정 주체) Domain Agent (HR 도메인 업무 처리) + System Agent (플랫폼 운영). Domain Agent는 검증된 업무부터 점진 구축.
Domain Agent HR 도메인 업무 직접 처리 예: 채용 서류심사, 업적평가 결과 분석, 임원 석세션 후보군 확정, 입과 안내
System Agent 플랫폼 백엔드 운영 (사용자 비노출) 예: 데이터 민감도 분류 Agent, 차폐 검사 Agent, 자산 추출 Agent
Skill 절차·기준 매뉴얼 (md) Lab이 정형화·등록한 작업 가이드. LLM에 컨텍스트로 주입. 사례가 쌓이면 신규 Skill로 승격.
Tool 결정적 코드 동작 임직원 정보 조회, 잔여휴가 계산, 출장비 계산 등. AX&CI Lab이 구현·유지보수, HR 담당자 누구나 추가 요청 가능.

데이터 종

용어본질설명
① 운영 데이터 (Master) 플랫폼 자체 운영용 마스터 로그인·권한 관리용 사용자 식별 정보, 조직·개인·발령 등.
② 표준 데이터 (Standards) 공식 정형 자료 인사 제도, 규정, 업무 프로세스(L4/L5/L6), 표준 양식. AX&CI Lab 운영.
③ 지식 데이터 (Knowledge) 검수 통과한 노하우 사례집 HR 담당자 사용 흔적이 누적·검수되어 형성. 조직×도메인 권한 단위 + 5단계 공개 범위 부착.

보안·권한

용어본질설명
개인식별정보 패턴 대조 (PII Pattern) 식별자 모양 자동 검사 사번 / 주민등록번호 / 이메일 / 전화번호 / 임직원 평가 코드 등을 정규식·키워드로 자동 검사. 발견 시 민감/기밀 등급 자동 후보.
권한 단위 룩업 (Permission Lookup) 권한 분류표 검색 사용자가 속한 조직×도메인 권한 단위(예: MX사업부 × 채용)에 대해 사전 라벨된 정책(기본 등급, 차폐 항목, Domain Experts)을 빠르게 끌어옴.
비식별화 게이트 (Anonymization Gate) 차폐 + 사내 재매핑 단일 통제점 민감 등급 사외 송출 시 임직원 식별자·부서명·금액 등을 마스킹/익명화. 매핑 테이블은 사내 보관. 응답 수신 시 사내 재매핑.
플랫폼 자체 감사 로그 (Application-side Audit) 모든 호출의 application 레이어 로그 모델 vendor 로그와 별개로 플랫폼이 직접 기록. 사내 LLM(로그 미저장)도 사외 LLM도 동일 포맷으로 추적.
Function Calling 패턴 사외 LLM은 도구 호출 의도만 표현 사외 LLM에게 Tool/Skill/Agent 직접 접근 불허. "Tool X를 인자 Y로 호출" 의도만 받고 실행은 플랫폼 내부.
권한 분류표 권한 단위별 정책 매트릭스 각 (조직 × 도메인) 셀에 데이터 접근 권한 / 기본 등급 / 차폐 정책 / Domain Experts 라벨링.

모델·인프라

용어본질설명
모델 슬롯 (BYOM 어댑터) 모델 인터페이스 추상화 사내 LLM(GAUSS), 사외 LLM(Chat GPT, Gemini), 신규 모델을 같은 인터페이스로 호출하게 만드는 변환층. 모델 교체 비용 ~0.
의미 검색 저장소 (임베딩 저장소) RAG용 벡터 DB 텍스트(Skill md, 지식 데이터, 표준 데이터)를 의미 벡터로 저장. 사용자 입력과 의미 유사한 자료를 검색해 LLM 컨텍스트로 첨부. 사내 임베딩 모델 사용 (한국어 + 보안).
의미 기반 검색 (RAG, Retrieval-Augmented Generation) 관련 자료 검색 후 답변 사용자 질의 → 의미 검색 저장소에서 관련 자료 검색 → LLM에 컨텍스트로 첨부 → 답변. 회사 맥락 반영 핵심.
GAUSS 사내 LLM 오픈소스 모델 파인튜닝 기반. 모델 가중치·추론·로그까지 사내 통제. vendor 로그 미저장 운영 가능.

자산화·검수

용어본질설명
Work Card 사용자 일감 단위 Guided Work Mode의 사용자 일감 단위. 클릭 시 L3-L6 후보와 Agent/Skill/Tool/Context 자동 조립.
Work-space 사용자가 보는 작업 환경 로그인 권한 기반 작업 환경. Open Chat과 Guided Work 진입을 함께 제공.
Open Chat Mode 채팅형 진입 완전 채팅창에서 시작. 플랫폼이 뒤에서 권한, 민감도, L3-L6 후보를 추정.
Guided Work Mode 추천 업무형 진입 반복·표준 업무를 Work Card로 제안. 사용자가 L6를 직접 찾지 않도록 설계.
검수 후보 (Tacit Candidate) 검수 전 자산 후보 카드 사용 중 발생한 결정·판단·반복 패턴이 자동 큐잉된 자유 텍스트 본문 + 메타데이터.
Domain Experts 권한 단위별 자산 검수자 부문/사업부에서 선별. 조직×도메인 권한 단위별로 지정. 주당 3-5시간 검수 (역할 정의·평가 연계).
지식 신뢰 단계 검수 상태 raw signal → tacit candidate → 검수 통과 지식. 공개 범위(개인/권한 단위/...)와 직교 축.
