Data Governance

HR 데이터를 어떻게 안전하게 다룰 것인가 — 보안 · 자산화 · 권한 관리 3대 정책 영역. 원문 HR Data/Knowledge는 전사 공통 AI Platform에 올리지 않고, HR AX가 AI Tool 앞단에서 등급·차폐·권한·감사를 통제한다.

1. 보안 — 안전한 관문

HR 데이터 그대로 사외 LLM 사용 시 임직원 개인정보·인사 결정 누설 우려. 사내망 개방으로 사외 LLM·사내 AI Tool 활용 환경 도래 → "막을 수 없으면 통제해야".

메커니즘 4단계

  1. 데이터 민감도 분류 Agent (사내 LLM 전담) — 개인식별정보(PII) 패턴 대조 + 권한 단위 룩업 + 사내 LLM 한국어 맥락 보강
  2. 등급 3단 라우팅 — 기밀 / 민감 / 일반
  3. 처리 경로 분기
    • 기밀 → 사내 LLM (GAUSS, 로그 미저장)
    • 민감 → 비식별화 게이트(차폐 + 사내 재매핑) → 사외 LLM
    • 일반 → 사외 LLM 그대로
  4. 플랫폼 자체 감사 로그 모든 호출 기록
체크포인트 › 기밀이거나 민감한 HR 데이터/정보가 외부로 나가지 않습니다

등급 3단 처리 매트릭스

등급처리모델
기밀 사외 송출 금지 사내 LLM (GAUSS, 로그 미저장)
민감 비식별화 게이트 후 외부 사외 LLM (Chat GPT / Gemini 등)
일반 그대로 외부 사외 LLM

2. 자산화 — 검수 기반 지식/노하우 축적

HR 담당자는 AI 빌더가 아님 → 도메인별 RAG·로컬 DB 직접 운영 불가. 완전 채팅형 진입만 있을 경우 업무 맥락과 검수 단위가 흐려질 수 있으므로, 플랫폼이 대화와 Work Card 양쪽에서 자산화 후보를 잡아야 함.

메커니즘 5단계

  1. Work Card 사용 흐름에서 흔적 자동 누적
  2. 중요 판단·결정·반복 패턴이 검수 후보로 자동 추출
  3. Domain Experts 검수 (승인 / 반려 / 보류)
  4. ③ 지식 데이터 등재 (공개 범위 부착)
  5. 반복 패턴은 Lab 결정으로 신규 L6 또는 신규 Skill 승격
체크포인트 › 사용할 수록 People팀만의 데이터·지식·노하우가 쌓입니다

두 가지 승격 경로

③ 지식 데이터 (사례 누적) | +-- 경로 A: 규정/기준/업무 프로세스 형태로 정형 가능 | → ② 표준 데이터 신규 L6 노드 추가 | → 드뭄 | +-- 경로 B: 절차·기준 매뉴얼로 정형 가능 → 신규 Skill 등록 → 잦음

자동 포착은 동일 메커니즘 (반복 패턴 감지). Lab이 4분기 결정: 새 L6 / 새 Skill / 둘 다 / 둘 다 아님.

3. 권한 관리 — 조직×도메인 권한 단위

부문 / 사업부 / 도메인별 데이터 민감도와 접근 범위가 모두 다름. 일반 RBAC만으로는 HR 도메인 권한 표현 불가.

메커니즘 3단계

  1. 권한 단위 = 조직 × 도메인 (예: MX사업부 × 채용)
  2. 권한 분류표 — DW 접근 / 기본 등급 / 차폐 정책 / Domain Experts 라벨링
  3. 지식 공개 범위 5단계 — 개인 / 권한 단위 / 도메인 종방향 / 조직 횡방향 / 전사 표준
체크포인트 › 필요한 사람에게, 필요한 데이터만, 필요한 목적에 한해 제공합니다

권한 단위 매트릭스

X축 (조직): AX&CI Lab(admin) / 부문 People팀 / 사업부 8개 (VD, DA, MX, NW, 의료기기, SR, 생기연, CDO)

Y축 (도메인 12개): 교육 / 노사 / 보상·근태 / 인력운영 / 임원조직 / 제도 / 채용 / 총무 / 해외인사 / DEI / 조직문화 / 집단지성

각 (조직 × 도메인) 셀에 4가지 라벨 부착:

  1. 데이터 접근 권한 — 어느 인사 DB 영역까지 열람 가능
  2. 기본 민감도 — 부문이라고 모두 High 아님. 권한 단위별 개별 라벨링.
  3. 차폐 정책 — 어느 항목을 어떻게 비식별화할지
  4. 자산 승급 검수자 — Domain Experts 지정

공개 범위 5단계

범위retrieval 가능자
개인작성자 본인
권한 단위같은 조직×도메인 권한 단위 담당자
도메인 종방향같은 도메인, 조직 계층 따라 (부문은 산하 사업부 자산 retrieval)
조직 횡방향같은 조직, 모든 도메인 (1차 출시 미지원)
전사 표준모든 권한 단위

원칙: 작성자 제안 + 검수자 확정 + 단계적 권한 escalation. 디폴트 좁게.
