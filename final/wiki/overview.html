한 장 요약 + 4 Phase Roadmap.

왜 HR은 독립 플랫폼인가

이유 1

HR 데이터 특수성

임직원 개인정보·평가·인사 결정은 사외 LLM에 직접 보낼 수 없고 전사 공통 AI Platform에도 올릴 수 없음. AI Tool 앞단의 안전 게이트가 필요.

이유 2

HR 도메인 권한 체계

부문 × 사업부 × 도메인 교차 권한은 일반 RBAC로 표현 불가. 권한 단위별 등급·차폐 정책·검수자가 모두 다름. HR 도메인 지식 없으면 정책 자체를 만들 수 없음.

이유 3

People 조직 자산화 책임

HR 담당자는 Light user(AI 빌더 아님). 도메인별 RAG·로컬 DB 직접 운영 불가. People팀이 단일 운영 책임을 가져야 자산이 누적됨.

HR AX 진행 Roadmap

Phase 1·2 산물 위에 Phase 3 플랫폼이 올라가고, Phase 4에서 AI Native 표준업무로 전면 도입.

Phase기간제목핵심 산물상태
Phase 1 '25.12 ~ '26.3 HR 업무 프로세스 / FTE 분석 L4-L6 업무 트리, AI 도입 후보 식별 완료
Phase 2 '26.3 ~ '26.6 교육 Quick-Win 과제 + 도메인별 Agent 개발 Top-down(도메인 주도) + Bottom-up(사업부 주도) 병행 진행 중
Phase 3 '26 하반기 HR AX 플랫폼 구축 Data Governance + Harness Engineering 두 축 예정
Phase 4 '27 AI Native 표준업무 프로세스 전면 도입 Digital Worker 우선 처리 업무 적용 예정

진입 방식 비교

진입 방식강점주의점플랫폼 처리
Open Chat Mode 완전 채팅창에서 바로 시작. 탐색형·신규·비정형 업무에 유리. 업무 맥락, 권한 범위, 민감도, 자산화 단위가 흐려질 수 있음. 입력 의도 분석 → 권한 확인 → L3-L6 후보 추정 → 필요 시 Guided Work로 전환.
Guided Work Mode 반복 업무, 표준 프로세스, 검수·감사·자산화가 필요한 업무에 유리. 초기 카드 설계가 과하면 사용자가 선택 피로를 느낄 수 있음. 로그인 권한·최근 업무·반복 패턴 기반으로 Work Card와 L6 후보를 먼저 제안.

두 모드는 경쟁 관계가 아니라 같은 통제 엔진을 공유하는 진입 방식이다. 어느 쪽으로 시작해도 권한 확인, 민감도 분류, 비식별화, 감사 로그, 검수형 자산화는 동일하게 적용된다.
