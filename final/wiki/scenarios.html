사용 시나리오

Work-space에서 Card 진입 → 자동 조립 → 처리 → 자산화의 흐름을 시나리오 단위로 풀어냄.

시나리오 A — 정형 카드 + Domain Agent 있음 기밀 등급

업무: 채용 직무적합성평가 (L3 채용 / L5 서류평가의 정형 카드)

흐름

[1] HR 담당자: Work-space 로그인 → 내 카드 목록 확인 [2] "신규 지원자 직무적합성평가" 카드 클릭 [3] 카드 진입 즉시 자동 조립 - Agent : 채용 직무적합성평가 Agent (Domain Agent) - Skill : "지원자 직무적합성 평가 기준" 매뉴얼 자동 부착 - Tool : 임직원 정보 조회기, 학력 검증기 - Context: 채용 도메인 표준 데이터 + 유사 지원자 케이스 (지식 데이터) [4] 데이터 민감도 분류 Agent 작동 - 결과: 등급 = 기밀 (지원자 개인정보 + 평가 의도) - 라우팅: 사내 LLM (GAUSS, 로그 미저장) [5] Agent 처리 - 지원자 서류 특이사항 하이라이트 - 직무 적합성 점수 + 근거 제시 [6] 사용자 검토 → 결정 ("OO 사유로 보류") - 카드 종료 [7] 시스템이 결정·판단 자동 추출 → 검수 후보 큐 진입 [8] Domain Experts(채용팀 선임 프로) 검수 → 승인 - ③ 지식 데이터 등재 (권한 단위 한정) [9] 다음 비슷한 지원자 카드 처리 시 이 사례가 retrieval로 부착됨

시나리오 B — 정형 카드 + Domain Agent 없음 민감 등급

업무: 신규 평가 제도 모판 작성 (정형 노드는 있지만 Domain Agent 미구축)

흐름

[1] HR 담당자: "평가 제도 모판 작성" 카드 클릭 [2] 카드 진입 시 자동 조립 (Agent 없이) - Skill : "평가 제도 모판 양식 매뉴얼" 자동 부착 - Tool : 표준 양식 템플릿 조회기 - Context: 인사 제도 표준 데이터 + 과거 모판 사례 - Agent : 미부착 → 기본 LLM 호출 [3] 데이터 민감도 분류 Agent 작동 - 결과: 등급 = 민감 (특정 임원 평가 사례 인용 가능) - 라우팅: 비식별화 게이트 → 사외 LLM [4] 비식별화 게이트 - 임원_A → "임원_X" 마스킹 - 부서명·금액 → 일반화 - 매핑 테이블은 사내 보관 [5] 사외 LLM 처리 (Skill 매뉴얼 + Context 주입) - 모판 초안 생성 [6] 응답 수신 → 사내 재매핑 (임원_X → 임원_A 복원) - 사용자 검토 → 결정 한 줄 입력 [7] 검수 후보 큐 진입 → Domain Experts 검수 - 승인 시 ③ 지식 데이터 등재 (도메인 종방향 후보) [8] 반복 패턴 누적 → Lab이 "평가 제도 모판 작성 Agent" 신규 구축 결정

시나리오 C — 비정형 카드 일반 등급

업무: 사외 시장 분석 정리 (트리 외 일감)

흐름

[1] HR 담당자: 새 카드 생성 → 도메인 "채용" 지정 → 자유 입력 "올해 IT 업계 채용시장 트렌드 정리해줘. 우리 직무군 중심으로." [2] 카드 진입 시 자동 조립 (비정형) - Skill : 채용 도메인 일반 매뉴얼 부착 - Context: 채용 도메인 표준 데이터 - Agent : 미부착 → 기본 LLM 호출 - Tool : 미부착 [3] 데이터 민감도 분류 Agent - 결과: 등급 = 일반 (공개 정보) - 라우팅: 사외 LLM 그대로 [4] 사외 LLM 처리 - 시장 트렌드 정리 + 직무군 매핑 [5] 사용자 검토 → 결정 한 줄 입력 "이 분석을 ZBR 자료에 반영" [6] 검수 후보 큐 진입 → Domain Experts 검수 - 자료 인용 / 직무군 매핑 패턴 등재 [7] 비정형 패턴 누적 → "사외 채용시장 트렌드 정리"가 반복되면 → 신규 L6 노드 후보 (정형화) → 또는 신규 Skill 매뉴얼 후보

시나리오 D — 인사위 안건 정리 기밀 등급

업무: 다음달 임원 인사위 안건 정리 (임원조직 도메인, L5 정형)

[1] HR 담당자: "임원 인사위 안건 정리" 카드 클릭 [2] 자동 조립 - Agent : 인사위 안건 정리 Agent (Domain Agent) - Skill : "인사위 안건 양식·기준" 매뉴얼 - Tool : 임원 인사 이력 조회기 - Context: 임원조직 표준 + 지난 안건 사례 (권한 단위 한정) [3] 데이터 민감도 분류 → 기밀 (임원 식별자 + 평가 이력) 라우팅: 사내 LLM (GAUSS, 로그 미저장) [4] GAUSS 응답 (사내 처리, 외부 송출 0) [5] 화면에 "임원 인사 데이터 감지 → 사내 LLM 처리, 외부 차단" 알림 [6] 사용자 검토 → 결정 입력 [7] 검수 후보 → ③ 지식 데이터 (권한 단위 한정)

업무 단위 카드
