사용 시나리오

Open Chat 또는 Guided Work로 진입 → 권한 확인 → L3-L6 후보 추정 → 자동 조립 → 처리 → 자산화의 흐름을 시나리오 단위로 풀어냄.

진입 모드

Mode사용자가 하는 일플랫폼이 하는 일적합한 업무
Open Chat 채팅창에 바로 업무 의도 입력 권한·민감도·L3-L6 후보를 추정하고 필요 시 카드형 흐름으로 전환 탐색형, 신규, 비정형, 아직 표준화되지 않은 일
Guided Work 추천 Work Card 선택 또는 최근 업무 재개 L3-L6, Agent, Skill, Tool, Context를 자동 조립 반복 업무, 표준 프로세스, 감사·검수·자산화가 필요한 일

시나리오 A — Guided Work + Domain Agent 있음 기밀 등급

업무: 서류 합격자 명단 확정 (L3 채용 / L4 선발전형 / L5 서류심사 / L6 서류 합격자 명단 확정)

흐름

[1] HR 담당자 로그인 → MX사업부 People팀 / 채용 권한 자동 확인 [2] Guided Work 추천 목록에서 "서류 합격자 명단 확정 지원" 카드 클릭 [3] 카드 진입 즉시 자동 조립 - L3/L4/L5/L6: 채용 / 선발전형 / 서류심사 / 서류 합격자 명단 확정 - Agent : 채용 서류심사 Agent - Skill : 서류심사 합격자 확정 기준 - Tool : 지원자 명단 추출기, 평가 결과 취합기, 채용Hub 조회기 - Context: 채용 표준 데이터 + 유사 합격/보류 케이스 [4] 데이터 민감도 분류 Agent 작동 - 결과: 등급 = 기밀 (지원자 개인정보 + 평가 의도) - 라우팅: 사내 LLM (GAUSS, 로그 미저장) [5] Agent 처리 - 합격/보류 후보 구분 - 보류 사유와 추가 확인사항 정리 - 현업 의견 충돌 지점 표시 [6] 사용자 검토 → 결정 ("OO 사유로 보류") - 카드 종료 [7] 시스템이 결정·판단 자동 추출 → 검수 후보 큐 진입 [8] Domain Experts(채용팀 선임 프로) 검수 → 승인 - ③ 지식 데이터 등재 (권한 단위 한정) [9] 다음 비슷한 지원자 카드 처리 시 이 사례가 retrieval로 부착됨

시나리오 B — Guided Work + Domain Agent 없음 민감 등급

업무: 업적평가 결과 분석 보고서 확정 (L3 제도 / L4 업적평가 / L5 결과 분석/보고 및 결과 안내 / L6 업적평가 결과 분석 보고서 확정)

흐름

[1] HR 담당자: "업적평가 결과 분석 보고서 초안" 카드 클릭 [2] 카드 진입 시 자동 조립 (Agent 없이) - L3/L4/L5/L6: 제도 / 업적평가 / 결과 분석/보고 및 결과 안내 / 업적평가 결과 분석 보고서 확정 - Skill : 업적평가 결과 분석 보고서 작성 기준 - Tool : 평가 분포 집계기, 보고서 템플릿 조회기 - Context: 제도 표준 데이터 + 과거 분석 보고서 사례 - Agent : 미부착 → 기본 LLM 호출 [3] 데이터 민감도 분류 Agent 작동 - 결과: 등급 = 민감 (개인 평가 결과와 조직별 분포 포함 가능성) - 라우팅: 비식별화 게이트 → 사외 LLM [4] 비식별화 게이트 - 임직원명 → "임직원_X" 마스킹 - 조직명·등급 분포 소수 셀 → 일반화 - 매핑 테이블은 사내 보관 [5] 사외 LLM 처리 (Skill 매뉴얼 + Context 주입) - 보고서 초안 생성 [6] 응답 수신 → 사내 재매핑 - 사용자 검토 → 결정 한 줄 입력 [7] 검수 후보 큐 진입 → Domain Experts 검수 - 승인 시 ③ 지식 데이터 등재 (도메인 종방향 후보) [8] 반복 패턴 누적 → Lab이 "업적평가 결과 분석 Agent" 신규 구축 결정

시나리오 C — Open Chat 일반 등급

업무: 채용시장 트렌드와 후보 채널 영향 정리 (자유 입력 → L3-L6 후보 추정)

흐름

[1] HR 담당자: Open Chat에 바로 자유 입력 "올해 IT 업계 채용시장 트렌드 정리해줘. 우리 직무군 중심으로." [2] 플랫폼이 업무 의도 분석 - 후보 1: 채용 / 인재발굴 / 인재 소싱 채널 선정 및 액팅 / 채널별 후보자 명단 취합 - 후보 2: 채용 / 채용계획 / 채용 계획 수립 / 부서별 채용 수요조사 취합 [3] Open Chat용 기본 조립 - Skill : 채용 도메인 일반 매뉴얼 부착 - Context: 채용 도메인 표준 데이터 - Agent : 미부착 → 기본 LLM 호출 - Tool : 미부착 [4] 데이터 민감도 분류 Agent - 결과: 등급 = 일반 (공개 정보) - 라우팅: 사외 LLM 그대로 [5] 사외 LLM 처리 - 시장 트렌드 정리 + 직무군 매핑 [6] 사용자 검토 → 결정 한 줄 입력 "이 분석을 ZBR 자료에 반영" [7] 검수 후보 큐 진입 → Domain Experts 검수 - 자료 인용 / 직무군 매핑 패턴 등재 [8] 비정형 패턴 누적 → 반복되면 → 신규 L6 노드 후보 (정형화) → 또는 신규 Skill 매뉴얼 후보

시나리오 D — 임원 석세션 후보군 명단 확정 기밀 등급

업무: 임원 석세션 후보군 명단 확정 (L3 임원조직 / L4 인력운영 / L5 임원 석세션 플랜 / L6 임원 석세션 후보군 명단 확정)

[1] HR 담당자: "임원 석세션 후보군 명단 확정" 카드 클릭 [2] 자동 조립 - L3/L4/L5/L6: 임원조직 / 인력운영 / 임원 석세션 플랜 / 임원 석세션 후보군 명단 확정 - Agent : 임원 석세션 플랜 Agent - Skill : 임원 석세션 후보군 확정 기준 - Tool : 임원 인사 원천 데이터 조회기, 후보군-포지션 매핑 조회기 - Context: 임원조직 표준 + 석세션 후보 평가 기준 + 보안문서 열람 이력 [3] 데이터 민감도 분류 → 기밀 (임원 식별자 + 평가·후보군·포지션 정보) 라우팅: 사내 LLM (GAUSS, 로그 미저장) [4] GAUSS 응답 (사내 처리, 외부 송출 0) [5] 화면에 "임원 석세션 데이터 감지 → 사내 LLM 처리, 외부 차단" 알림 [6] 사용자 검토 → 결정 입력 [7] 검수 후보 → ③ 지식 데이터 (권한 단위 한정)

공통 메시지

단일 진입

채팅 또는 카드

사용자는 채팅으로 시작하거나 추천 카드로 들어갈 수 있음.

자동 조립

Agent · Skill · Tool · Context

플랫폼이 L3-L6 후보와 적합한 인프라를 자동 부착. 사용자가 빌더가 될 필요 없음.

자산화

검수 통과 → 다음 사용

카드 종료 시 결정·판단이 검수를 거쳐 다음 카드로 흘러감.
