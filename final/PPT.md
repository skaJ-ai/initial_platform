## 4P. 통합 HR AX Platform 구축(Phase 3) - Data Governance

**[부제]** HR 데이터를 어떻게 안전하게 다룰 것인가 — 보안 · 자산화 · 권한 관리 3대 정책 영역

```
┌────────────┐  ┌────────────┐  ┌────────────┐
│ 1. 보안    │  │ 2. 자산화  │  │ 3. 권한 관리│
│ 안전한 관문 │  │ 지식/노하우│  │ 조직×도메인 │
│            │  │ 축적       │  │ 권한 단위   │
└────────────┘  └────────────┘  └────────────┘
```

### 1. 보안 — 안전한 관문

**메커니즘 4단계**
1. 데이터 민감도 분류 Agent (사내 LLM 전담)
2. 등급 3단(기밀 / 민감 / 일반) 라우팅
3. 기밀 → 사내 LLM(GAUSS, 로그 미저장) | 민감 → 비식별화 게이트(차폐 + 사내 재매핑) | 일반 → 사외 LLM 그대로
4. 플랫폼 자체 감사 로그 모든 호출 기록

**체크포인트** › 기밀이거나 민감한 HR 데이터/정보가 외부로 나가지 않습니다

### 2. 자산화 — 검수 기반 지식/노하우 축적

**메커니즘 5단계**
1. Work Card 사용 흐름에서 흔적 자동 누적
2. 중요 판단·결정·반복 패턴이 검수 후보로 자동 추출
3. Domain Experts 검수 (승인 / 반려 / 보류)
4. ③ 지식 데이터 등재 (공개 범위 부착)
5. 반복 패턴은 Lab 결정으로 신규 L6 또는 신규 Skill 승격

**체크포인트** › 사용할 수록 People팀만의 데이터·지식·노하우가 쌓입니다

### 3. 권한 관리 — 조직×도메인 권한 단위

**메커니즘 3단계**
1. 권한 단위 = 조직 × 도메인 (예: MX사업부 × 채용)
2. 권한 분류표 — DW 접근 / 기본 등급 / 차폐 정책 / Domain Experts 라벨링
3. 지식 공개 범위 5단계 — 개인 / 권한 단위 / 도메인 종방향 / 조직 횡방향 / 전사 표준

**체크포인트** › 필요한 사람에게, 필요한 데이터만, 필요한 목적에 한해 제공합니다

**[푸터]** HR 기밀/민감 데이터를 중심으로 내·외부 AI Tool을 보다 잘 활용할 수 있도록 설계

## 5P. 통합 HR AX Platform 구축(Phase 3) - Harness Engineering

**[부제]** LLM이 HR 맥락 위에서 정확히 일하도록 — Context · 실행 · 연계를 고려한 인프라 설계

```
┌────────────────┬─────────────────┬───────────────┐
│ 1. Context 공급 │ 2. 실행체계 운영 │ 3. 외부 연계  │
│ HR 맥락을      │ Agent · Skill · │ 시스템 · AI   │
│ LLM에 공급      │ Tool 3계층 운영 │ 서비스 허브   │
└────────────────┴─────────────────┴───────────────┘
```

### 1. Context 공급 — HR 맥락을 LLM에 공급

- **Process** — L3-L6 업무 프로세스 트리 (Phase 1 산물 + 운영 중 신규 L6 추가)
- **인사 규정** — HR 제도, 규정, 취업규칙 등
- **인사 기준** — 평가·근태·보상 기준, 표준 양식 등
- **노하우(사례)** — 지식 데이터. 검수 통과한 판단·결정·예외 케이스

### 2. 실행체계 운영 — Agent · Skill · Tool 3계층

**Agent**
본질: LLM 처리 단위 (판단·결정의 주체)
- **Domain Agent** — HR 도메인 업무 직접 처리. 검증된 업무부터 점진 구축. 없는 카드는 Skill·Tool만으로도 처리 가능.
  - 예: 채용 직무적합성평가, 임원인사 근황 작성, 제도 모판 제작
- **System Agent** — 플랫폼 백엔드 운영 (사용자 비노출)
  - 예: 데이터 민감도 분류, 차폐 검사, 자산 추출

**Skill**
본질: 절차·기준 매뉴얼 (md)
설명: Lab이 정형화·등록한 작업 가이드. 사례가 충분히 쌓이면 신규 Skill로 승격.

**Tool**
본질: 결정적 코드 동작
설명: 정확한 동작 (조회·계산·연동). Agent가 있으면 Agent가 호출, 없으면 Skill 매뉴얼 지시에 따라 호출.

### 3. 외부 연계 — 시스템 · AI 서비스 허브

- **인사 시스템 / Data Warehouse 연계**
- **사내 LLM (GAUSS)** — 기밀 데이터 처리
- **사외 LLM (Chat GPT, Gemini 등)** — 민감 데이터(비식별화 후), 일반 데이터 처리
- **사내 AI Tool (SPICA, AX Hub 등)** — 플랫폼 권한·차폐 정책 통과 시 연계

**[중앙 흐름]** HR 담당자 → Open Chat 또는 Guided Work로 진입 → L3-L6 후보 자동 추정 → 3대 인프라 자동 조립 → 처리

**[푸터]** HR 담당자는 L6를 직접 고르지 않음. 인프라 자동 조립 + 외부 시스템·AI 연계는 플랫폼이 처리

## 7P. 별첨 2. 플랫폼 세부 구현(안)

### 5개 레이어 구성

```
┌────────────────────────────────────────────────────────┐
│ [Layer 1] 사용자 진입                                   │
│   Work-space + Open Chat / Guided Work                 │
├────────────────────────────────────────────────────────┤
│ [Layer 2] 안전 게이트 (보안 핵심)                       │
│   ├─ 데이터 민감도 분류 Agent (사내 LLM 전담)           │
│   ├─ 비식별화 게이트 (차폐 + 매핑 사내 보관 + 재매핑)    │
│   ├─ Function Calling Broker (사외 LLM은 의도만 표현)   │
│   └─ 플랫폼 자체 감사 로그 (모든 호출 기록)             │
├────────────────────────────────────────────────────────┤
│ [Layer 3] 실행 (Agent · Skill · Tool)                  │
│   ├─ Domain Agent (검증된 업무에 점진 구축)             │
│   ├─ System Agent (분류 · 차폐 · 자산 추출)            │
│   ├─ Skill Registry (코드 리포 + DB 메타·임베딩)        │
│   └─ Tool 카탈로그 (Lab 구현, 담당자 추가 요청)         │
├────────────────────────────────────────────────────────┤
│ [Layer 4] 데이터 (3종)                                 │
│   ├─ ① 운영 데이터 (Master) — 로그인·권한·마스터        │
│   ├─ ② 표준 데이터 (Standards) — 제도·기준·L4/L5/L6     │
│   └─ ③ 지식 데이터 (Knowledge) — 검수 통과 노하우       │
├────────────────────────────────────────────────────────┤
│ [Layer 5] 외부 연계                                    │
│   ├─ 인사 시스템 / Data Warehouse                      │
│   ├─ 사내 LLM(GAUSS) — 기밀 응답·임베딩·분류 보강       │
│   ├─ 사외 LLM(Chat GPT/Gemini 등) — Function Calling   │
│   └─ 사내 AI Tool (SPICA, AX Hub 등) — 정책 통과 시 연계│
└────────────────────────────────────────────────────────┘
```

### 핵심 컴포넌트 요약

| 컴포넌트 | 용도 |
|---|---|
| 분류 Agent | 등급 자동 판정 — 개인식별정보 패턴 대조 + 권한 단위 룩업 + 사내 LLM |
| 모델 슬롯 (BYOM 어댑터) | GAUSS / 사외 LLM 같은 인터페이스로 호출, 모델 교체 비용 ~0 |
| 비식별화 게이트 | 민감 등급 차폐·재매핑, 매핑 테이블 사내 전용 |
| 의미 검색 저장소 (임베딩) | 지식 retrieval, 한국어 임베딩 (사내) |
| 플랫폼 자체 감사 로그 | 모든 호출 application-side 기록 |
| Review Queue | 지식 검수 워크플로우 — 승인/반려/보류 + 30일 자동 폐기 |

### 1차 출시 핵심 범위

권한 단위 N개 우선 적용 → 동작 검증 후 확대.
등급 분류 · 라우팅 · 비식별화 · 검수형 자산화 · 플랫폼 자체 감사 로그 5종 통합 동작.

## 8P. 별첨 3. 플랫폼 아키텍처 및 권한 관리 체계

### 한 장 아키텍처

```
[사용자 = HR 담당자]
       │
       │  로그인 → 조직/담당 도메인/권한 단위 자동 확인
       ▼
┌──────────────────────────────────────────────────────┐
│           HR AX Platform (안전 게이트)                │
│                                                       │
│   Work-space / Open Chat / Guided Work                │
│         ↓                                             │
│   데이터 민감도 분류 Agent                             │
│         ↓                                             │
│   ┌── 기밀 ──┐  ┌── 민감 ──┐  ┌── 일반 ──┐            │
│   │ GAUSS    │  │ 비식별화 │  │ 사외 LLM │            │
│   │ 로그미저장│  │ 게이트   │  │ 그대로   │            │
│   └─────────┘  └─────────┘  └─────────┘              │
│         ↓           ↓            ↓                   │
│   Function Calling Broker (Tool 실행은 사내)          │
│         ↓                                             │
│   플랫폼 자체 감사 로그                                │
│                                                       │
│   ① Master | ② Standards | ③ Knowledge                │
└──────────────────────────────────────────────────────┘
       ↑                                  ↑
       │ DW에서 필요 범위만                 │ 검수 후보 → 승인/반려/보류
       │                                  │
   [DW]                              [Domain Experts]
   GHRP / 삼성U / 채용Hub
```

### 권한 관리 체계

**X축 — 조직**

| 레벨 | 주체 | 열람 범위 |
|---|---|---|
| L0 Admin | AX&CI Lab | 전사 운영·표준 관리 |
| L1 부문 | 부문 People팀 | 자기 부문 + 산하 사업부 |
| L2 사업부 | 사업부 담당자 | 자기 사업부만 |

사업부 8개: VD / DA / MX / NW / 의료기기 / SR / 생기연 / CDO

**Y축 — 도메인 12개**
교육 / 노사 / 보상·근태 / 인력운영 / 임원조직 / 제도 / 채용 / 총무 / 해외인사 / DEI / 조직문화 / 집단지성

**권한 단위 라벨링 (권한 분류표)** — 각 (조직 × 도메인) 셀에 4가지:
1. 데이터 접근 권한
2. 기본 민감도 (권한 단위별 개별)
3. 차폐 정책
4. 자산 승급 검수자 (Domain Experts)

**지식 공개 범위 5단계**: 개인 / 권한 단위 / 도메인 종방향 / 조직 횡방향 / 전사 표준
원칙: 작성자 제안 + 검수자 확정 + 단계적 권한 escalation. 디폴트 좁게.
