# HR AX Platform — One Pager

---

## 한 줄 정의

**HR AX Platform은 HR 데이터가 AI Tool로 가기 전에 거치는 HR 특화 안전 게이트 + LLM 활용 인프라입니다.**
HR Data/Knowledge가 외부에 유출될 위험 없이 사내외 LLM과 효과적으로 동작할 수 있도록 운영할 수 있습니다.

---

## 두 축

### 🛡️ Data Governance — 데이터를 어떻게 안전하게 다룰 것인가
- **보안**: 데이터 등급 3단(기밀/민감/일반) 자동 분류 → 라우팅. 기밀은 사내 LLM(GAUSS, 로그 미저장), 민감은 비식별화 게이트 후 사외 LLM, 일반은 그대로.
- **자산화**: HR 담당자 사용 흔적이 검수를 거쳐 People 조직 지식 데이터로 누적. Domain Experts 검수 + 30일 미보강 자동 폐기.
- **권한 관리**: 조직×도메인 권한 단위(예: MX사업부 × 채용)로 DW·표준·지식 데이터 접근. 5단계 공개 범위(개인/권한 단위/도메인 종방향/조직 횡방향/전사 표준).

### ⚙️ Harness Engineering — LLM이 HR 맥락 위에서 정확히 일하도록
- **Context 공급**: Process(L3-L6) + 인사 규정 + 인사 기준 + 노하우(지식 데이터)
- **실행체계 운영**: Agent(Domain + System) · Skill(절차·기준 매뉴얼) · Tool(결정적 코드)
- **외부 연계**: 인사 시스템·DW · 사내 LLM(GAUSS) · 사외 LLM(Chat GPT, Gemini) · 사내 AI Tool(SPICA, AX Hub)

---

## 왜 HR은 독립 플랫폼인가 (3가지 이유)

1. **HR 데이터 특수성** — 임직원 개인정보·평가·인사 결정은 사외 LLM에 직접 보낼 수 없고 전사 공통 AI Platform에도 올릴 수 없음. AI Tool 앞단의 안전 게이트가 필요.
2. **HR 도메인 권한 체계** — 부문 × 사업부 × 도메인 교차 권한은 일반 RBAC로 표현 불가. HR 도메인 지식 없으면 정책 자체를 만들 수 없음.
3. **People 조직 자산화 책임** — HR 담당자는 Light user(AI 빌더 아님). 도메인별 RAG·로컬 DB 직접 운영 불가. People팀이 단일 운영 책임을 가져야 자산이 누적됨.

## HR 일하는 방식의 변화 (Before / After)

| 항목 | Before | After |
|---|---|---|
| 사외 LLM 사용 | 임직원 정보 그대로 입력 → 보안 위험 상존 | 자동 등급 분류 → 비식별화 게이트 통과 후 안전 사용 |
| 콜드 스타트 | 매 세션 빈 채팅창에서 처음부터 | Work-space 카드 진입 시 회사 컨텍스트 + 지난 판단 자동 부착 |
| 노하우 | 담당자 머릿속·개인 메모 | 검수형 자산화 → People 조직 지식 자본 |
| 권한 | 일반 RBAC, 도메인 정책 따로 | 조직×도메인 권한 단위 + 5단계 공개 범위로 일원화 |
| 후임 인계 | 인수인계 문서 의존 | 권한 단위 자산이 후임에게 자동 retrieval |

## 메시지 한 줄

**"AX 추진의 데이터 민감 영역을 풀어내는 People팀의 단일 책임 플랫폼."**
사외 LLM·사내 AI Tool은 활용 대상이지만, HR 도메인 정책·민감도 분류·자산화는 People팀이 책임지고 운영합니다.
