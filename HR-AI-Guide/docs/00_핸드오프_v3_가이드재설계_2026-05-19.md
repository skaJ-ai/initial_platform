### 1-3. HR 기밀 데이터 — 2종 (절대 확대 금지)

> ⚠️ **메모리·CLAUDE.md 합의사항**: 기밀 2종 확대 절대 금지. 건강·M&A·핵심인재 등을 RED로 옮기는 권고는 모두 *철회됨*. 회사 전략 = "기밀 최소화 + 권한관리 단위 최소화". LLM의 역할은 분류 *변경*이 아니라 분류에 따른 *행동 가이드*.

### 1-4. 권한 분리
- **기밀 취급자(부문 People 등 노사 관련)**: Enterprise LLM 사용 권한 *자체* 없음
- **일반 HR 담당자(사업부 HR 등)**: 사용 권한 있음. **본 가이드의 대상**
- 다만 기밀 비취급자도 우연히 노사 주제 입력 시도 가능 → 가이드가 통제 역할

### 1-5. 로그·모니터링 (J 정보 — 중요)
- Enterprise LLM 사용 로그는 **정보보호센터 지정 담당자(서약서 기반)**만 열람
- AX팀도 못 봄
- Enterprise 약관상 **외부 학습 미사용, 보안 철저** 보장
- → Workday 감사로그 수준에 가까운 통제

### 1-6. 회사 핵심 메시지 (변경 금지)
> "쓰지 말라가 아니라 안전하게 쓰시라"
- "본인 책임" 톤으로 가지 않음. **회사 + 본인 공동 책임** 모델.
- 위험 과장하지 않음 — 사용 위축 = 실패.
- 금지 일변도 아님 — 가공·일반화 후 활용 권장.

## 2. 이번 세션의 핵심 분석 통찰 (가이드 톤 설계에 직결)

### 2-1. Workday vs Enterprise LLM — 격차는 처음 생각보다 좁다
J 정보(로그 통제 + Enterprise 약관 + 인사관리 목적 한정)를 반영하면, Workday와의 통제 격차는 다음 정도로만 남는다:

| 차원 | J 정보 반영 후 상태 |
|---|---|
| 위탁 통제 | Workday와 등가 (Enterprise 약관 + 학습 미사용) |
| 로그 가시성 | Workday 감사로그 수준 (지정 담당자만, 서약서) |
| 사고 회복성 | 약관상 삭제 요청 가능 — 격차 미미 |
| **입력 단계 권한 통제** | **여전히 다름** — Workday는 화면 RBAC, LLM은 입력자가 권한 외 정보도 손으로 입력 가능 |
| **데이터 흐름 자유도** | **여전히 다름** — LLM 답변이 메일·문서로 흘러감 |
| **출력 비결정성** | **여전히 다름** — 환각·정보혼합 위험 |
| **목적 한정성** | 정책으로 보완 가능 |

→ **결론**: "Workday는 OK인데 LLM은 왜 가공"이라는 사용자 의문이 정당하다. 격차를 과장하지 말 것.

### 2-2. 동의 확대 시 마스킹이 *모두* 불필요한가 — 아니다, 다만 약화된다

법무가 동의 확대 인정해도 **3가지 항구적 마스킹 근거**가 남는다:

1. **다른 임직원·외부인 정보 보호** — 본인 동의 ≠ 타인 동의. 평가 코멘트·면접 메모·360 피드백 등 HR 데이터 대부분이 다중 주체.
2. **민감정보 별도 동의(§23)** — 동의 확대로 자동 포함되지 않는 별도 트랙
3. **미공개 중요정보(자본시장법)** — 임원·M&A·조직개편. 동의·개보법과 무관

→ 본인 정보에 대한 마스킹은 *법적 의무* → *모범 실무 권고*로 톤다운 가능. 위 3가지는 동의 무관 항구적 유지.

### 2-3. 민감정보(§23) — HR 실무에서 의미 있는 건 사실상 "건강 묶음"

법 범주: 건강·성생활 / 노조·정당 / 사상·신념·정치적 견해 / 유전정보 / 범죄경력 / 인종·민족 / 생체인식

HR 실무 의미: **건강 관련(병가·휴직·진단·산재·장애·임신·출산)** + 노조(이미 RED)

**핵심**: 민감정보는 "이름만 바꾸면 충분"이 아니다.
- 작은 집단에 분포 → **2개 속성만 결합해도 식별** (장애 + 부서, 임신 + 팀 등)
- 정보 *자체*가 차별·낙인 위험 — 식별자 가려도 위험 남음
- 제3자 정보가 자주 끼어듦
- §23 입법 취지 자체가 더 보수적 처리 요구

→ 민감정보 가공 규칙은 "**카테고리 자체를 일반화하거나 입력에서 빼기**". 직책·부서와 결합 자체를 피함.

### 2-4. "3+ 속성 결합 금지" 휴리스틱의 한계
- 모집단 크기·속성 희소성에 따라 무너짐
- 가이드에 넣을 가치는 있지만 **유일한 규칙으로 강조 금지**
- **특수 결합 금지 리스트**와 병행 필요:
  - 임원·소수직책 + 어떤 속성이든
  - 소수자 보호계층(임신·육아휴직·장애·병가) + 부서/팀
  - 평가 하위·징계 + 부서/팀
  - 소수 조직(전사 5명 이하) + 어떤 속성이든

### 2-5. 다른 기업 벤치마크 (애플류 "고신뢰 + 개인 책임" 모델)
- 단순 원칙 4~5개로 끝, 마스킹 매트릭스 없음
- 작동 조건: 기술 백본(아키텍처 격리) + 인재 선발 + 명확한 결과 + 무처벌 보고 문화
- **J 회사 적용 불가** — HR 스킬 분포 넓음, 백본 부분 커버, 회사 메시지가 "공동 책임"
- 흡수할 정신: **가이드 분량 압박** + **원칙 우선** + **에스컬레이션 잘 보이게** + **사고 보고 처벌 분리**

### 2-6. 복잡한 마스킹 가이드의 실효성 — 솔직히 낮다
- 비기술 HR이 Level 1/2/3 구분하며 한 단어씩 가공할 가능성 ≈ 1%
- 두 행동만 발생: (a) 무시 + 원문 입력, (b) 부담돼서 LLM 미사용. 둘 다 실패
- **진짜 무기 = 시나리오별 안전 프롬프트 템플릿** (복붙으로 끝)
- 마스킹은 "원칙·거버넌스용"으로 두되, 실무 메인은 템플릿으로 전환

### 5-2. 2단계 — 가공 압축 + 시나리오 템플릿 (변경사항 8~10)
