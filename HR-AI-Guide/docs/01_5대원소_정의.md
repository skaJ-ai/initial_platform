# 5대 데이터 원소 — 정식 정의 (합의 완료)

> 이 문서는 가이드의 지적 골격입니다. **변경 금지.**
> Lab원 지적("공통 기준에서 출발해야 한다")을 반영하여 도출.

---

## 📚 개요

모든 HR 데이터를 5개 *원소 유형*(A~E)으로 분해하고, 각 원소의 처리 규칙을 명확히 합니다.
이것이 모든 카테고리에 일관 적용되는 *공통 기준*입니다.

카테고리(임원/조직, 평가, 보상 등)는 *예시 모음*일 뿐이며, 각 카테고리는
"이 카테고리에는 보통 어떤 원소가 섞이는지" 보여주는 역할을 합니다.

---

## 🧬 5대 원소

### A. 직접 식별자 (Direct Identifier)

**정의**: 그 자체로 한 사람을 가리키는 정보

**포함**: 이름, 사번, 이메일, 휴대전화, 사진, 사원증 번호, 직원 SNS 핸들, 좌석번호

**처리 원칙**: 항상 *제거* 또는 *가명 대체* (예: "직원 A")

**예외 없음**: Level 1부터 모든 Level에서 항상 처리

**예시**:
- 김민수 → 직원 A
- sm.kim@company.com → [이메일]
- 010-1234-5678 → [휴대전화]

**비고**: 주민·외국인·여권번호 등 고유식별정보는 RED 자체이므로 입력 금지 (제거 처리가 아예 일어나지 않음).

---

### B. 조합 식별자 (Quasi-Identifier)

**정의**: 단독은 약하지만 조합 시 특정 가능한 속성

**포함**: 부서명, 직책·직급, 연차, 성별, 연령대, 학교명, 전 직장명, 가족구성, 거주지역, 차량번호, 자격증 종류

**처리 원칙**: 조합 식별성 검토 후 *일반화*
- 단독은 무해해도 조합 3~4개가 모이면 특정 가능
- "직군 단위", "○○대급", "30대" 등 한 단계 위 범주로

**판단 기준**: "이 조합으로 우리 회사에서 떠오르는 사람이 5명 이하라면 식별 가능"

**예시**:
- "마케팅2팀 30대 여성 차장" → "마케팅 직군 차장급"
- "○○대 경영학과 ○○년 졸업" → "4년제 경영계열 졸업"

---

### C. 판단 값 (Evaluative Value)

**정의**: 개인에 부착된 정량·정성 평가 결과

**포함**: 평가등급(S/A/B/C), 점수, 순위, KPI 달성률, 핵심인재 등급(HiPo), 평판조회 결과, 면접 점수, 승진 추천 순위, **연봉 자체** (단 연봉의 시점·맥락은 D)

**처리 원칙**:
- 정량값 → 구간화 ("4.8점" → "평균 상회")
- 정성값 → 경향 표현 ("탁월/평이/미흡" → "긍정적/중립/개선 필요")
- 순위 → 그룹화 ("3위" → "상위권")

**주의**: 판단 값은 *A/B와 결합되면 위험도 폭증* — A+C 또는 B+C 결합 시 즉시 처리

**예시**:
- 평가 A등급 → 평가군 상위
- 4.8점 → 평균 상회
- 3위 → 상위권

---

### D. 맥락 식별자 (Contextual Identifier)

**정의**: 시·공간·금액·상대방 등 사건을 특정하는 좌표

**포함**: 회사명·거래상대방, 프로젝트명, 구체 일정(YYYY.MM.DD), 구체 금액, 부서 신설/통합 방향, 임원 일정, 회의 장소, M&A 거래구조, **연봉 협상 시점·맥락**

**처리 원칙**: 시·공간·금액·상대방 모두 *추상화*
- 회사명 → "동종업 1개사" / "외부 협력사"
- 일정 → "향후 일정" / "최근"
- 금액 → "검토 중인 보상안" / 범위 표시

**특수성**: D 원소는 *조합 식별을 우회*합니다 — 이름을 가려도 "○년 ○월 X사와의 M&A 협상"이라는 좌표만으로 사건 특정. 그래서 Level 3가 필요한 이유.

**예시**:
- 2026.7.1 X사 인수 협의 → "향후 동종업 인수 협의"
- 구체 금액 ○○억 → "보상 검토안"

---

### E. 민감 사실 (Sensitive Fact)

**정의**: 그 사실 자체가 법적·사회적으로 민감

**포함**: 건강·질병·장애, 정신건강·상담 이력, 가족사정(이혼·사별·중증환자), 종교, 정치성향, 노조가입*, 징계·고충·괴롭힘 사안**, 성적지향, 임신·출산·육아 사정

**처리 원칙**: *원칙적 제거*, 필요시 *범주화*
- "우울증 진단 후 휴직" → "건강 사유 휴직"
- "이혼 후 양육비 부담" → "가정 사정"

**비고**:
- *노조 가입은 RED — 처리 대상이 아니라 입력 금지*
- *건강 정보는 RED 준용 — 별도 명시 (산업안전보건법 제132조)*
- *징계·고충 사안은 YELLOW 고위험, Level 3 적용 + AX&CI Lab 문의 권장*

---

## 🎚 Level 1 / 2 / 3 — 원소 기반 정의

| Level | 명칭 | 처리 대상 원소 |
|---|---|---|
| **Level 1** | 기본 마스킹 | A만 |
| **Level 2** | 익명처리 | A + B + C |
| **Level 3** | 맥락 추상화 | A + B + C + D, **E는 원칙 제거** |

### 카테고리별 Level 매핑

- **Level 1**: 10. 총무 (일부)
- **Level 2**: 5. 인사기본 / 6. 성과평가 / 8. 보상근태 / 9. 채용(일반) / 10. 총무(민감 부분)
- **Level 3**: 3. 임원/조직 / 4. M&A / 7. 핵심/리텐션 / 9. 채용(임원영입·평판) / 노무·징계 사안

### 이 정의의 장점

- "Level 3가 왜 더 강한가?" → 「D 맥락 식별자까지 처리하기 때문」 (느낌이 아닌 명확한 정의)
- 새로운 데이터 만나도 「어느 원소가 들어있나?」 만 보면 Level 결정
- 카테고리에 없는 데이터(예: 외부 협력사 직원 평가)도 원소 분해로 대응 가능
- Lab원·실무자 모두 동일 언어로 판단

---

## 🔑 핵심 원칙

1. 모든 HR 데이터는 5개 원소(A~E)의 조합으로 분해할 수 있다.
2. 데이터 처리는 「어느 원소까지 처리하느냐」로 결정되며, 이것이 Level 1/2/3의 본질이다.
3. 카테고리(임원/조직, 평가 등)는 「어떤 원소가 주로 섞이는지」를 알려주는 예시 모음일 뿐이다.
4. 새로운 데이터를 만나면 「이 안에 어떤 원소가 섞였지?」를 먼저 물어본다.
5. E 원소(특히 건강 정보)는 RED 준용으로 별도 관리한다.
