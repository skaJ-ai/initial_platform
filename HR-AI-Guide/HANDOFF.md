## 2. 사용자 요구와 배경

목표는 Enterprise LLM 오픈 전 HR 담당자에게 배포할 사용 가이드를 HTML로 만드는 것이다.

핵심 배경:

- 정보보호센터는 2026년 6월 중 고유식별정보, API key 등 전사 공통 위험정보 12종을 필터링하는 솔루션 도입 예정.
- HR 데이터에는 고유식별정보 외에도 임직원 개인정보와 민감 맥락이 많다.
- AX팀과 협의된 기준은 “기밀 데이터를 제외하고는 Enterprise LLM 사용 가능”이다.
- HR 기밀 데이터는 현재 2종으로 정리됨:
  - 고유식별정보
  - 노사/협의회 관련 자료
- 고유식별정보는 전사 솔루션으로 차단될 가능성이 높지만, 노사/협의회 관련 자료는 자동 차단되지 않을 가능성이 높다.
- 노사/협의회 관련 데이터를 취급하는 일부 부문 People 인력은 Enterprise LLM 사용 권한 자체가 없지만, 사업부 HR 담당자는 사용 권한이 있을 수 있다.
- 임직원 개인정보의 Enterprise LLM 활용 동의 범위는 법무 검토 중이다. Enterprise LLM 오픈이 법무 검토보다 앞설 수 있으므로, 검토 완료 전까지는 개인정보 원문 입력을 피하고 가공 후 사용하도록 안내해야 한다.

## 3. 현재 반영한 정책 판단

중요한 설계 결정:

1. “기밀 2종 제외 전부 사용 가능”이라는 문구는 그대로 쓰지 않았다.  
   이유: 배포문으로는 너무 넓고, 개인정보 법무 검토가 끝나지 않은 상태와 충돌한다.

2. 현재 가이드는 아래 3단계로 정리했다.
   - `RED / 입력 금지`: 고유식별정보, 노사/협의회 자료, 개인 계정/미승인 AI 사용, AI에게 인사결정 자체를 위임하는 행위
   - `YELLOW / 가공 후 사용`: 임직원·지원자 개인정보, 평가, 보상, 채용, 휴직·건강·가족, 징계·고충·조사, 임원·조직·M&A 등
   - `GREEN / 사용 가능`: 회사·개인·조직·사건 식별정보가 없는 일반 HR 업무 질의

3. 다른 LLM 리서치 문서에는 계좌번호, 의료·건강 상세 정보, 법적 분쟁 자료도 RED로 들어가 있었지만, 최종 HTML에서는 AX팀 합의 기준과 맞춰 대부분 `YELLOW / 고위험 개인정보` 또는 `문의` 성격으로 조정했다.

4. “익명처리”라는 표현은 일부 기존 문맥에는 있었지만, 새 v1에서는 가능한 한 `마스킹`, `가명처리`, `범주화`, `맥락 추상화`, `가공` 중심으로 썼다. 법적 의미의 익명처리는 기준이 높으므로 남용하지 않는 편이 좋다.

5. “법무 검토를 거쳐 적법성을 확보하였다” 같은 확정 문구는 제거했다. 현재 상태는 “법무 검토 중이며, 완료 전까지 원문 개인정보 입력을 피한다”가 맞다.

## 4. HTML 구조

파일: `deliverables\HR_AI_가이드_v1_초안.html`

주요 섹션:

- `#search`: 검색창과 빠른 검색 칩
- `#principle`: 핵심 원칙, RED/YELLOW/GREEN 카드, 자동 필터링 솔루션의 한계
- `#checklist`: LLM 입력 전 자가점검
- `#masking`: 개인정보 가공 기준
- `#examples`: 업무별 빠른 기준과 프롬프트 변환 예시
- `#incident`: 실수로 입력했을 때 대응 절차
- `#appendix`: 사용 권한, 전사 보안가이드, 법무 검토 전 임시 기준

검색 로직:

- `guideData`: 검색 결과 카드 데이터
- `quick`: 빠른 검색 칩 목록
- `scoreCard()`: 검색어와 카드 키워드를 비교해 점수화
- `renderCard()`: 결과 카드 렌더링
- `fallback()`: 매칭 실패 시 “판단 불가” 카드 표시

검색 점수식은 exact match를 우선하도록 조정했다. 예를 들어 `징계`는 `징계·고충·조사 사안`으로, `징계 수위`는 `AI에게 인사결정 자체를 위임`으로 매칭된다.
